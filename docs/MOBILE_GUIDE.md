# Руководство по мобильной версии

## Обзор

Приложение полностью оптимизировано для мобильных устройств:
- ✅ Адаптивный дизайн (mobile-first)
- ✅ Правильное масштабирование для всех размеров экранов
- ✅ Оптимизированная навигация
- ✅ Сжатые таблицы и карточки на мобиле

---

## Точки разрыва (Breakpoints)

Приложение использует стандартные Tailwind breakpoints:

| Размер | Ширина | Устройство |
|--------|---------|-----------|
| `xs` | < 640px | Мобильные телефоны |
| `sm` | ≥ 640px | Планшеты в портрете |
| `md` | ≥ 768px | Планшеты в пейзаже |
| `lg` | ≥ 1024px | Ноутбуки |
| `xl` | ≥ 1280px | Десктопы |

## Примеры использования

### 1. Адаптивный размер шрифта

```tsx
// На мобиле text-base, на больших экранах text-lg
<h1 className="text-base sm:text-lg font-bold">Заголовок</h1>
```

### 2. Скрытие/Показ элементов

```tsx
{/* Видимо только на мобиле */}
<div className="sm:hidden">Мобильный контент</div>

{/* Видимо только на больших экранах */}
<div className="hidden sm:block">Десктопный контент</div>

{/* На мобиле карточки, на больших экранах таблица */}
<div className="sm:hidden">Карточки</div>
<div className="hidden sm:block">Таблица</div>
```

### 3. Адаптивные отступы

```tsx
{/* px-3 на мобиле, px-4 на больших */}
<div className="px-3 sm:px-4">Контент</div>

{/* py-3 на мобиле, py-4 на больших */}
<div className="py-3 sm:py-4">Контент</div>
```

### 4. Адаптивные размеры

```tsx
{/* w-12 на мобиле, w-16 на больших */}
<img className="w-12 sm:w-16" />

{/* text-sm на мобиле, text-base на больших */}
<p className="text-sm sm:text-base">Описание</p>
```

## Тестирование на разных размерах

### Chrome DevTools
1. Откройте DevTools (F12)
2. Нажмите Ctrl+Shift+M или кнопку "Toggle device toolbar"
3. Выберите разные устройства из выпадающего списка

### Рекомендуемые размеры для тестирования
- **iPhone SE** (375×667)
- **iPhone 12 Pro** (390×844)
- **iPad** (768×1024)
- **Laptop** (1366×768)

### Что проверить
- [ ] Таббы видны и кликабельны на мобиле
- [ ] Текст заголовков читается полностью (не обрезается)
- [ ] Таблица превращается в карточки на мобиле
- [ ] Нет горизонтального скролла (кроме таблиц при необходимости)
- [ ] Кнопки достаточно большие для тача (не менее 44×44px)
- [ ] Модальные окна видны полностью на мобиле

## Проблемы и решения

### Проблема: Таббы обрезаются на узком экране

**Решение:** Используется горизонтальный скролл:
```tsx
<div className="flex overflow-x-auto scrollbar-none">
  {/* Таббы */}
</div>
```

### Проблема: Таблица выходит за границы экрана

**Решение:** На мобиле показываем карточки вместо таблицы:
```tsx
{/* Мобильная версия */}
<div className="sm:hidden">Карточки</div>

{/* Десктопная версия */}
<div className="hidden sm:block overflow-x-auto">Таблица</div>
```

### Проблема: Кнопки слишком малые для тача

**Решение:** Минимальный размер кнопки 44×44px:
```tsx
<button className="px-4 py-2 min-h-[44px]">Кнопка</button>
```

### Проблема: Модальное окно не влезает на экран

**Решение:** Добавить max-height и scrollbar:
```tsx
<div className="max-h-[90vh] overflow-y-auto">
  {/* Контент модального окна */}
</div>
```

## Best Practices

### 1. Используйте mobile-first подход
```tsx
// ❌ Неправильно
<div className="sm:px-4 px-2">Контент</div>  // сначала десктоп

// ✅ Правильно
<div className="px-2 sm:px-4">Контент</div>  // сначала мобиль
```

### 2. Скрывайте неважную информацию на мобиле
```tsx
<td className="hidden sm:table-cell">Колонка</td>
```

### 3. Используйте карточки вместо таблиц на мобиле
```tsx
{/* На мобиле - карточки */}
<div className="sm:hidden space-y-2">...</div>

{/* На десктопе - таблица */}
<div className="hidden sm:block">...</div>
```

### 4. Тестируйте на реальных устройствах
Эмулятор - это хорошее начало, но реальные устройства могут вести себя по-другому.

### 5. Минимальный размер сенсорных целей
Кнопки и ссылки должны быть не менее 44×44px для удобства на мобиле.

## Производительность на мобиле

### Оптимизация изображений
```tsx
{/* Загружайте меньшие изображения на мобиле */}
<img 
  src="small.jpg" 
  className="w-12 sm:w-24"
  alt="Логотип"
/>
```

### Минимизация DOM
```tsx
{/* Не рендеры скрытые элементы */}
{/* ✅ Правильно */}
{isMobile && <MobileComponent />}
{!isMobile && <DesktopComponent />}

{/* ❌ Неправильно */}
<div className="hidden sm:block"><DesktopComponent /></div>
```
